Naming convention:

  • Program
    • When used in a name "program" refers to the application, local config and related files on the local machine that is being supported by GUPdotNET.
    • For example GUPProgramConfig.xml contains information about the application GUPdotNET is being used to update like Name, Title, Version, etc.
  • Package
    • When used in a name "package" refers to remote application, config and releted file concerning the newest application version that can be upgraded to, files are located on a server and accessed via http.
      • More access methods (ftp, web service, etc.) may be made available later.
    • For example GUPPackageConfig.xml contains information on each OS, installer type and processor update that is currently available.
